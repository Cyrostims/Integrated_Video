A Behind the Curtains Look into My Profile


This is the primary index.html page that runs the portfolio.
As you can see it's quite simple overall.
The bulk of the heavy lifting is done by the javascript file.

This is the css for my portfolio.
I primarily used bootstrap 3.2.0 and then adjusted it as needed.

This is the index.js page for my items that create an array where I declare the images and assign them to the proper item module file.

This is the index.js page for my pages and function the same as the items index.js page does.

This is the about.md pages file. A very straightforward file that uses markdown for the writing syntax.

These combine to make the index.js file which does the heavy lifting for my profile. Within the code, I make different classes to handle different aspects. Becareful of naming conventions as a spelling mistake can be a time consuming error to fix when trying to link all the classes together.


The overall idea is that I generate an object/array that I then populate with the module files. However, it's not just a single object/array as the items and pages are handled separately from one another.

The final step I took was publishing it to Github and using their pages function to host my portfolio.
